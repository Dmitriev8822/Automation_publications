# AGENTS.md

Инструкции для агентов, работающих с репозиторием `Automation_publications`.

## Общий порядок работы

1. Прочитать `doc`.
2. Прочитать `docs/ARCHITECTURE.md`.
3. Прочитать свой файл задачи в `docs/tasks/`.
4. Проверить существующие файлы целевого модуля.
5. Реализовывать только свой модуль и необходимые тесты.
6. Не менять публичные контракты без обновления `docs/ARCHITECTURE.md` и связанных задач.
7. Запустить релевантные тесты.
8. Обновить документацию своего модуля в `docs/modules/<module>.md`.

## Правила интерфейсов

- Общие сущности импортировать из `app.schemas`.
- Настройки получать через `app.config.get_settings()` или принимать объект `Settings` снаружи.
- В тестах использовать fake/mock, а не реальные внешние сервисы.
- `app/service.py` должен оставаться единственным местом бизнес-процесса публикации.
- `app/scheduler.py` должен только запускать job по расписанию.
- `app/main.py` должен только собирать зависимости и запускать приложение.

## Документация модулей

Каждый реализованный модуль обязан иметь файл `docs/modules/<module>.md` со структурой:

```markdown
# Модуль <name>

## Назначение

## Публичные классы и функции

## Используемые настройки

## Взаимодействие с другими модулями

## Обработка ошибок

## Тестирование

## Пример использования
```

## Запреты

- Не добавлять реальные секреты в репозиторий.
- Не делать реальные API-вызовы в unit-тестах.
- Не смешивать ответственность модулей.
- Не менять имена публичных методов без обновления документации.
- Не оставлять модуль без `docs/modules/<module>.md`.
