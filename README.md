# Automation_publications

MVP-сервис для автоматической подготовки и публикации постов в Telegram-канал.

## Идея

Приложение по расписанию находит свежие новости по заданной теме, пропускает уже опубликованные источники, генерирует Telegram-пост, при необходимости готовит изображение, публикует результат в канал и сохраняет факт публикации в SQLite.

## Структура

```text
app/
  ai.py          # интерфейс AI-клиента и каркас OpenRouter-адаптера
  config.py      # настройки из .env и переменных окружения
  database.py    # SQLAlchemy-модель и репозиторий публикаций
  main.py        # точка входа приложения
  scheduler.py   # периодический запуск публикации
  schemas.py     # общие Pydantic-контракты
  service.py     # основной бизнес-сценарий
  telegram.py    # интерфейс Telegram-публикатора

docs/
  ARCHITECTURE.md
  tasks/         # ТЗ для независимой разработки модулей
  modules/       # документация реализованных модулей

tests/           # быстрые тесты без реальных внешних API
```

## Быстрый старт

1. Установить зависимости:

```bash
python -m pip install -r requirements.txt
```

2. Создать локальный `.env` на основе примера:

```bash
cp .env.example .env
```

3. Запустить тесты:

```bash
pytest -q
```

4. Запустить приложение после реализации реальных OpenRouter и Telegram-адаптеров:

```bash
python -m app.main
```

## Текущий статус

Создана основная структура модулей и стабильные интерфейсы между ними. Реальные HTTP-вызовы OpenRouter и Telegram намеренно оставлены как отдельные задачи, чтобы следующие агенты могли реализовывать модули независимо по документам из `docs/tasks/`.
