# Документация модулей

В этой директории агенты должны создавать итоговую документацию после реализации своих модулей.

Для каждого модуля нужен отдельный файл:

- `schemas.md`;
- `config.md`;
- `database.md`;
- `ai.md`;
- `telegram.md`;
- `service.md`;
- `scheduler.md`;
- `main.md`;
- `tests.md`.

## Обязательная структура файла модуля

```markdown
# Модуль <name>

## Назначение

## Публичные классы и функции

## Используемые настройки

## Взаимодействие с другими модулями

## Обработка ошибок

## Тестирование

## Пример использования
```
