# Задача 04. Модуль Telegram

## Цель

Реализовать публикацию сгенерированного поста и изображения в Telegram-канал через `pyTelegramBotAPI`.

## Работать в файлах

- `app/telegram.py`
- `app/config.py` при необходимости новых настроек
- `app/schemas.py` при необходимости согласования типов
- `tests/test_telegram.py`
- `docs/modules/telegram.md`

## Что реализовать

1. Класс `TelegramPublisher`.
2. Метод:
   - `publish_post(post: GeneratedPost, image: ImageAsset | None = None) -> int`.
3. Если `image` передан, публиковать изображение с подписью.
4. Если `image` не передан, публиковать текстовый пост.
5. Возвращать `telegram_message_id`.
6. Использовать настройки:
   - `TELEGRAM_BOT_TOKEN`;
   - `TELEGRAM_CHANNEL_ID`.

## Интерфейсы для других модулей

`service.py` ожидает такой контракт:

```python
message_id = telegram_publisher.publish_post(generated_post, image)
```

Если публикация невозможна, метод должен выбросить исключение с понятным сообщением.

## Откуда брать информацию

- Настройки: `app/config.py`.
- Общие схемы: `app/schemas.py`.
- Документация `pyTelegramBotAPI`.
- Архитектура: `docs/ARCHITECTURE.md`.

## Что нельзя делать

- Не генерировать текст поста в `telegram.py`.
- Не искать новости в `telegram.py`.
- Не сохранять статус публикации в БД напрямую.
- Не выполнять реальные Telegram API-запросы в unit-тестах.

## Тесты

Проверить с fake/mock bot:

- публикацию текстового поста;
- публикацию поста с изображением;
- возврат `message_id`;
- проброс ошибки Telegram;
- отсутствие реальных HTTP-запросов.

## Документация после реализации

Создать `docs/modules/telegram.md` и описать:

- как создать бота;
- как добавить бота администратором канала;
- какие настройки нужны;
- как работает публикация текста и изображения;
- как тестировать модуль без реального Telegram.
