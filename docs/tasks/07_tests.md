# Задача 07. Общая стратегия тестирования

## Цель

Собрать быстрый набор тестов, который проверяет основные модули и может запускаться при старте приложения без реальных внешних API.

## Работать в файлах

- `tests/`
- `pytest.ini` или `pyproject.toml`, если будет выбран один из этих файлов
- `docs/modules/tests.md`

## Что реализовать

1. Настроить pytest.
2. Разделить тесты на быстрые unit-тесты и потенциальные integration-тесты.
3. Быстрые тесты должны проходить без:
   - реального OpenRouter;
   - реального Telegram;
   - сетевых запросов;
   - production-БД.
4. Добавить fake/mock-объекты для:
   - AIClient;
   - TelegramPublisher;
   - PostRepository или временной БД.
5. Обеспечить запуск командой:

```bash
pytest
```

## Минимальное покрытие

Проверить:

- схемы и валидацию;
- конфигурацию;
- БД и защиту от дублей;
- AI-модуль с mock-ответами;
- Telegram-модуль с fake bot;
- полный сервисный сценарий;
- scheduler/main без бесконечного запуска.

## Откуда брать информацию

- `docs/ARCHITECTURE.md`.
- Все документы `docs/tasks/*.md`.
- Документация модулей `docs/modules/*.md` после реализации.

## Что нельзя делать

- Не использовать реальные ключи.
- Не отправлять сообщения в настоящий Telegram-канал.
- Не делать реальные запросы в OpenRouter.
- Не делать тесты зависящими от порядка запуска.

## Документация после реализации

Создать `docs/modules/tests.md` и описать:

- какие тесты есть;
- какие fake/mock используются;
- как запускать быстрые тесты;
- какие тесты запрещено включать в startup-проверку.
